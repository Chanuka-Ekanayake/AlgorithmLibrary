# Treap (Cartesian Tree)

The Treap data structure cleanly unites strict configurations of a **Binary Search Tree** with a **Max-Heap**, enforcing simple rotational boundaries governed by uniformly distributed random node priority rules to dynamically maintain stability.

## Why use Treaps?
Tree layouts like the AVL and Red-Black tree use complicated path-height detection conditions to constantly readjust variables safely without breaking parent relationships. With a Cartesian priority system wrapped securely inside each internal block, mathematical entropy naturally minimizes heavy branch depth errors. This allows it to comfortably deliver reliable self-balancing limits within $O(\log n)$ constraints statically over tens of thousands of edits dynamically.

## Quick File Structure
1. `core/treap.py`: The executable python structure hosting class mechanics.
2. `docs/`: Markdown notes specifying the core rules on structural bubbling bounds and time complexities.
3. `test-project/`: Easy-to-run system diagnostics testing the system via simulated usage models.
