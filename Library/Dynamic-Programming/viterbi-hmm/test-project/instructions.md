# Viterbi Decoder Test Suite Instructions

This directory contains `app.py`, an executable interactive script to prove the validity of the core `ViterbiDecoder` logic built in `core/decoder.py`.

## Running the Application

Executions of the algorithm should be done from the command line interface utilizing Python 3. The system relies strictly on Standard Libraries, eliminating the need for `pip install` setups or heavy virtual environments (like Conda).

### MacOS / Linux

1.  Open your Terminal application.
2.  Navigate your directory path to this specific algorithm folder:
    ```bash
    cd path/to/AlgorithmLibrary/Library/Dynamic-Programming/viterbi-hmm/
    ```
3.  Execute using the Python interpreter:
    ```bash
    python3 test-project/app.py
    ```

### Windows Context

1.  Open Command Prompt or PowerShell.
2.  Navigate to the directory using Windows backslashes as path separators:
    ```cmd
    cd path\to\AlgorithmLibrary\Library\Dynamic-Programming\viterbi-hmm\
    ```
3.  Deploy using standard Python alias:
    ```cmd
    python test-project\app.py
    ```

## Detailed Evaluation of Outputs

When the application runs, it initiates two distinct mathematical scenarios proving that the Viterbi Decoder correctly leverages the Markov Property.

### Output Segment 1: The Weather Prediction Scenario

The first component generated by the test application revolves around predicting the weather based upon a friend's activities context.

**Expected Sandbox Output:**
```text
============================================================
SCENARIO 1: WEATHER DEDUCTION HMM
============================================================
Observation Sequence: ['walk', 'shop', 'clean']
Most likely weather sequence: ['Sunny', 'Rainy', 'Rainy']
Calculated Sequence Probability: 0.01344000
------------------------------------------------------------
```

**What is happening?**
1.  On day 1, the friend went for a `walk`. The decoder evaluates the starting probabilities: `Sunny` is 40% but emits `walk` 60% of the time (Score = 0.24). `Rainy` is 60% but emits `walk` only 10% of the time (Score = 0.06). Thus, day one is mapped as `Sunny`.
2.  On day 2, the friend `shop`s. `Sunny`->`Sunny` transition is high, but `Sunny` emits `shop` at only 30%. However, `Sunny`->`Rainy` transition is decent (40%) and `Rainy` emits `shop` relatively frequently (40%). The path weight for `Sunny`->`Rainy` combined ends up overcoming passing to `Sunny`->`Sunny`. Day 2 flips to `Rainy`!
3.  On day 3, the friend `clean`s. Since the friend is far more likely to clean on a rainy day (50%), and the system is currently `Rainy`, it prefers the `Rainy`->`Rainy` loop.

### Output Segment 2: Natural Language Processing (POS Tagging)

The second output component mimics foundational NLP algorithms. Part-Of-Speech tagging relies heavily on context clues. For instance, the word `bear` acts fundamentally differently in the sentence `I saw a bear` vs `I bear a heavy burden.`.

**Expected Sandbox Output:**
```text
============================================================
SCENARIO 2: PART OF SPEECH TAGGING HMM
============================================================
Observation Sequence: ['they', 'hunt', 'bear']
Predicted POS Tags:   ['Noun', 'Verb', 'Noun']
Calculated Sequence Probability: 0.01470000

Observation Sequence: ['bear', 'is', 'funny']
Predicted POS Tags:   ['Noun', 'Verb', 'Adj']
Calculated Sequence Probability: 0.03136000
------------------------------------------------------------
```

**What is happening?**
Look closely at the word `bear`. 
- In the first sentence, `they` strongly resolves as a **Noun**. Finding a `hunt` following a Noun strongly projects to be a **Verb**. Because verbs transition heavily into objects (**Noun**s), the Viterbi decoder correctly identifies `bear` here as an animal Object (**Noun**).
- In the second sentence, `bear` is the first variable. It's likeliest to be a **Noun**. The subsequent `is` acts overwhelmingly as a **Verb**. Verbs transition well into **Adjectives**, identifying `funny` successfully. The DP cache perfectly handles context-based routing.

## Extending the Sandbox

If you wish to test your own unique algorithms mathematically inside this layout:
1. Open `test-project/app.py` in your IDE.
2. Build a new `def run_my_example():` function block mirroring `run_weather_example()`.
3. Create generic `states` arrays and generic `observations` arrays.
4. Design mathematically sound matrices where probabilities map to $1.0$.
5. Pass them dynamically to `ViterbiDecoder` logic execution method and trace. Ensure to append it to the `main()` function execution tree! 

Note again that floating point multiplication natively suffers from underflow issues over $T > 100$ string arrays without algorithmic Log-Space conversions. The `app.py` module limits input variables to strictly $T<50$ instances dynamically avoiding zero-drop precision bugs locally.
