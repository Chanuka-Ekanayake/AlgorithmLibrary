# User Guide: Automated Product Tiering (K-Means)

This interactive simulation demonstrates how **Unsupervised Machine Learning** is used in e-commerce and SaaS platforms to automatically categorize products based on technical specifications, without the need for manual human tagging.

## Environment Setup

Before running the simulation, ensure you have the necessary mathematical library installed. This project uses **NumPy** for high-performance vectorized operations.

1. **Install Dependencies**:
```bash
pip install numpy

```


2. **Navigate to the Directory**:
```bash
cd categories/machine-learning/k-means-product-clustering/test-project

```


3. **Launch the Simulator**:
```bash
python app.py

```

## Learning Exercises

Use the following scenarios to observe how the K-Means algorithm "thinks" and organizes your marketplace.

### Scenario A: The "Two-Tier" Budget Strategy

When you start the app, enter `2` for the number of clusters.

* **Observation:** The algorithm will divide your catalog into two broad groups: "Basic Utilities" and "Heavyweight AI." This is useful for simple filtering.

### Scenario B: Granular Market Segmentation

Run the app again and enter `4` for the number of clusters.

* **Observation:** Notice how the algorithm isolates the "Extreme" models (like GPT-4-Turbo) into their own cluster, separate from "Mid-Range" tools (like Random Forest). This allows for highly targeted pricing tiers.

### Scenario C: Predict a New Product

After the clustering finishes, use the "Prediction" feature to input your own custom model specs.

* **Exercise:** Enter a Complexity of `5.0` and a Compute Demand of `5.0`.
* **Result:** See which "Business Tier" the algorithm assigns your hypothetical product to based on the previously calculated centroids.

## Engineering Insights: Why K-Means?

As a Software Engineer, why would you implement this instead of using a series of `if/else` statements?

1. **Adaptability:** If your catalog grows from 10 models to 10,000, K-Means will automatically adjust the tier boundaries based on the *actual* distribution of the data, rather than relying on hard-coded thresholds.
2. **Multi-Dimensionality:** While this demo uses 2 features (Complexity and Compute), K-Means can easily handle 50+ features (Price, RAM usage, Latency, etc.) simultaneously—something impossible for a human to visualize or hard-code.
3. **Performance:** By using the optimized implementation in `core/kmeans.py`, the categorization happens in milliseconds, making it suitable for dynamic backend updates.

## Data Format

The simulation uses a numerical representation of software. In a production environment, you would use **Model Embeddings** (vectors generated by an LLM) as the input for this clustering logic.

| Attribute | Range | Description |
| --- | --- | --- |
| **Logic Complexity** | 1.0 - 10.0 | Higher values indicate complex neural architectures. |
| **Compute Demand** | 1.0 - 10.0 | Higher values indicate heavy GPU/RAM requirements. |
